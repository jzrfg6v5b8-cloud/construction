# 疑难排查

## 插件未出现

确认扩展管理器中已启用、`.rbz` 支持当前 SketchUp 主版本并完成重启。查看 Ruby 控制台中的首个加载错误；不要重复安装多个同名版本。企业签名策略拦截时联系管理员发布受信包。

## JSON 被拒绝

根据错误码检查 UTF-8、schema、单位、项目/方案/revision、文件大小、稳定 ID、有限数值和资源 SHA-256。不要删除校验字段来“通过”。绝对路径、`..`、网络路径和符号链接逃逸应始终被拒绝。

## 比例或位置错误

先停止同步。核对 `units`、Z-up/右手坐标、项目原点、角度单位和变换顺序；检查组件内部轴及是否存在嵌套/非均匀缩放。不要用整体缩放目测修正尺寸。

## 组件缺失或重复

核对 `componentId + libraryVersion`、文件哈希和批准别名表。清理仅按显示名匹配的旧组件；检查采购根组件规则，避免嵌套子件重复统计。占位体必须保持醒目标识并阻断批准。

## SKU 统计不一致

在模型保存后重新全量统计，检查隐藏归档、排除标签、嵌套采购根、成套计量、空/TBD SKU、房间归属和非均匀缩放。输入 JSON 的数量不能替代模型重算。

## 本地桥接连接失败

确认服务只监听回环地址，双方协议版本一致且已重新配对。检查 Origin 白名单、短时令牌、系统代理/防火墙和端口占用。不要改成 `0.0.0.0`、通配 CORS 或长期令牌来排障。

## 同步反复产生同一差异

比较输入 SHA-256、稳定 ID、归一化单位、浮点容差和模型中保存的源摘要。若插件/组件库版本改变，先完成迁移；不要靠重复点击应用消除差异。

## LayOut 显示旧模型

在 `.layout` 的文档引用中确认路径和 SHA-256 对应 handoff manifest，手动刷新 `.skp`。检查视口绑定的是明确场景而非最后保存视图。插件无法从 SketchUp 侧保证 LayOut 已刷新。

## 关联尺寸错位或警示

停止导出。确认引用的是正确修订，逐条检查尺寸吸附点；模型几何被替换后即使视觉位置相近，关联也可能失效。重新关联并对照复尺资料人工签署，不能批量忽略警示。

## PDF 导出异常

由用户在 LayOut 检查页面范围、纸张、字体、视口渲染、磁盘空间、文件权限和 DRAFT/批准状态，再点击导出。仓库和插件都不提供自动 PDF；SketchUp Ruby API 不能直接完成 LayOut 全流程。

## 提交诊断

提供错误发生时间、可复现步骤、错误码、SketchUp/LayOut/插件/schema/组件库版本、operation ID 和脱敏日志。不要提交访问令牌、完整客户 JSON、真实绝对路径或未授权模型。
