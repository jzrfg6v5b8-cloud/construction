# 安装与使用

## 前置条件

- 使用 handoff manifest 指定的 SketchUp/LayOut 同一主版本；跨主版本另存前先复制项目包。
- 仅从内部发布渠道取得签名或带 SHA-256 的 `.rbz`，核对发布说明和哈希。
- JSON、组件库和项目文件放在本地受控目录，不从邮件临时目录直接运行。

## 安装

1. 在 SketchUp 打开“扩展程序管理器”；
2. 选择“安装扩展程序”，选取已验证的 `.rbz`；
3. 重启 SketchUp，在扩展列表中确认发布者、版本和启用状态；
4. 打开插件“关于/诊断”，确认插件版本、支持的 SketchUp 主版本和数据 schema 版本。

不要把 Ruby 文件随意复制到 Plugins 目录来绕过来源校验。企业策略阻止未签名扩展时，应由管理员发布受信版本，不应关闭全局安全策略。

## 首次配置

- 选择项目工作区和只读组件库目录；
- 若启用本地桥接，仅允许回环地址、随机高位端口和短时令牌，详见[桥接安全](bridge-security.md)；
- 关闭“打开文件即自动写入”；默认先校验并显示差异；
- 设置自动快照目录，但不要把客户数据写入公共同步盘。

## 日常使用

1. 在 Web 系统导出同一方案版本的 JSON 和资源，核对 `projectId`、`schemeId`、`revision`、单位及哈希；
2. 在 SketchUp 新建/打开项目副本，运行“校验并预览导入”，解决 blocking 问题后再应用；
3. 人工检查尺寸、原点、楼层、标签、组件方向、碰撞、材质、场景和 SKU 统计；
4. 保存 `.skp`，生成 handoff manifest，并按模板文件名放入项目包；
5. 最后在 LayOut 完成“打开并刷新—检查—点击导出”三步。

导入成功仅表示数据通过格式校验并写入 SketchUp，不表示设计、复尺、合规或 LayOut 已批准。插件不提供二进制 `.layout`，也不自动产出最终 PDF。

## 卸载/升级

升级前保存模型副本和当前 manifest。通过扩展管理器禁用旧版本，安装经验证的新版本，重启后先用非生产样例做迁移预检。若 schema 或属性字典版本变化，必须先完成迁移报告，再打开正式项目。
