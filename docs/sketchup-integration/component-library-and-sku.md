# 组件库与 SKU 统计

## 组件库约定

每个批准组件具有不可变的 `componentId`、规范化 SKU、组件库版本、尺寸、单位、轴向、插入点、允许缩放规则、供应商来源和文件 SHA-256。文件名可读但不作为身份；同名组件不得自动视为同一 SKU。

- 原点放在约定落地点，Z 轴向上，正面方向统一；
- 实例默认缩放为 `[1,1,1]`；非均匀缩放须阻断或作为明确变体；
- 动态组件、材质和嵌套组件必须说明统计规则；
- 只有经尺寸校验的模型可参与精确碰撞；占位体标记 `representation=placeholder`；
- 废弃组件保留映射和迁移说明，不原地覆盖旧版本文件。

解析顺序为稳定 `componentId + libraryVersion`，再核对 SKU 和哈希。找不到、哈希不符或尺寸超容差时使用带醒目标识的占位体，并阻断批准交付。

## SKU 规范化

SKU 去除首尾空白并统一约定大小写，但不得擅自删除内部连字符、前导零或把供应商别名合并。别名只能来自已审核映射表。空 SKU、`TBD`、自由文本名称和材料色号不计为已绑定 SKU。

## 统计口径

- `instanceCount`：遍历模型中参与交付的叶级采购实例；排除隐藏归档、辅助几何和模板图签；
- `uniqueSkuCount`：对规范化且已批准 SKU 去重；
- `unboundInstanceCount`：应采购但没有批准 SKU 的实例数；
- `quantity`：通常每实例 1；成套/包装/延米/面积必须由明确计量字段决定，不能仅按实例猜测；
- 同一实例只由最接近的“采购根组件”计数，避免嵌套椅腿、五金重复累计。

统计结果按 SKU、房间、组件库版本列出，并包含排除数量和原因。场景可见性不改变全模型采购数量；页面统计若按场景过滤，必须显式标注范围。

## 对账

将模型重算结果与批准 SKU 台账和 handoff manifest 比较。任一未绑定、未知别名、重复稳定 ID、非均匀缩放、数量单位冲突或输入/模型计数不一致均为 blocking。A03023 示例故意保留一个未绑定实例，用于演示门禁。

LayOut 中的 SKU 表仍需在刷新模型后人工检查；插件统计不会自动刷新 LayOut 表格或自动导出 PDF。
