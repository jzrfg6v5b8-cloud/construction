# 本地桥接安全

本地桥接是 Web 应用与 SketchUp 插件之间的可选消息通道，不是公网 API。可离线导入 JSON 时优先离线方式。

## 最小安全基线

- 仅绑定 `127.0.0.1`/`::1`，启动时选择随机高位端口；不得绑定 `0.0.0.0`。
- 每次会话生成至少 128 位随机令牌，短时有效、仅内存保存；令牌不进入 URL、日志、manifest 或模型属性。
- 请求使用 `Authorization: Bearer`，校验精确 `Origin` 白名单、`Host`、方法和 `Content-Type`；禁止通配 CORS。
- 首次连接显示双方验证码/配对确认；项目切换、重启、超时或多次失败后令牌失效。
- 每个写请求带 request ID、时间戳、nonce 和正文摘要；拒绝过期、重复、乱序或摘要不匹配的请求。
- 限制正文大小、请求速率、并发数和 JSON 深度；只接受固定 schema，不接受任意命令、Ruby、文件 URL 或 shell 参数。
- TLS 不能自动消除本机恶意进程风险；回环桥接的核心是短时令牌、Origin 校验、用户确认和最小能力。

## 能力分级

`status:read`、`import:preview`、`import:apply`、`handoff:write` 分开授权。状态读取不能隐式升级为模型写入。`import:apply` 必须在 SketchUp 内显示差异并由用户确认；桥接不能在 SketchUp 不可见时后台覆盖模型。

## 文件边界

桥接只传结构化消息或项目工作区内的相对资源引用。解析规范化后的真实路径必须仍位于批准根目录；拒绝路径穿越、符号链接逃逸、UNC/网络共享、设备文件和超出白名单的扩展名。资源使用 SHA-256 校验后再打开。

## 日志与响应

日志保留时间、operation ID、插件/协议版本、动作、结果和对象计数；屏蔽令牌、Authorization、完整 JSON、客户姓名和本地绝对路径。错误响应不返回堆栈和文件系统细节。安全事件包括失败配对、重放、Origin 不符、超限和路径拒绝。

## 停止与恢复

SketchUp 退出或插件禁用时立即关闭监听器。检测到异常请求时暂停写能力、撤销当前 operation、轮换令牌并要求重新配对；不要自动重试写请求。

桥接不控制 LayOut，也不能绕过用户执行刷新引用、检查关联尺寸和点击导出的三步。
