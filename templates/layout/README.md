# LayOut 交付模板说明

本目录只提供可审计的模板约定和 JSON 示例，**不包含二进制 `.layout` 模板，也不会自动生成 PDF**。团队应在所用 SketchUp/LayOut 主版本中人工建立并保存 `.layout` 文件。

## 建议目录

```text
<project>/
├── <project>_<scheme>_<revision>.layout
├── handoff-manifest.json
├── assets/
│   ├── models/<project>_<scheme>_<revision>.skp
│   └── references/
└── exports/
```

`.layout`、`.skp` 和导出文件不得嵌入仓库示例；交付时放入项目包。模型引用优先使用项目包内相对位置，移动项目包后必须重新检查引用。

## 母版与页面

在 LayOut 中创建 A3 横向母版，统一页边距、标题栏、页码、项目号、方案号、修订号、日期、绘图/复核人和“DRAFT/复尺后施工”等状态。建议页面：

1. `A-000` 封面与说明；
2. `A-101` 尺寸平面，1:50；
3. `A-102` 家具平面，1:50；
4. `A-201` 立面索引，1:50；
5. `A-301...` 房间立面，1:20；
6. `A-401` 节点与产品标注，1:10 或 1:5；
7. `A-501` SKU/材料表；
8. `A-901` 校核、变更和免责声明。

页面编号保持稳定；删页应保留变更记录，不复用已发布编号。

## 视口、场景与比例

- 每个 LayOut 视口只绑定一个明确命名的 SketchUp 场景，禁止依赖“最后保存视图”。
- 场景名采用 `<sheet>-<purpose>-<view>`，例如 `A-101-DIM-PLAN`、`A-301-LIVING-NORTH`。
- 场景应保存相机、可见标签、剖切面、样式和阴影状态。更新模型后不要在 LayOut 内临时改视角来“修图”。
- 正交图使用正投影和固定比例；透视图标注 `NTS`，不得显示为可量取比例图。
- 常用比例：总平/家具平面 `1:50`，房间立面 `1:20`，节点 `1:10`/`1:5`。视口裁切、纸张或比例变化后重新核对标注。
- 视口设置为高质量矢量/混合渲染前先验证性能与线型；最终导出前检查栅格化和字体替换。

## 关联尺寸

关联尺寸必须吸附到 SketchUp 模型几何，而不是视口边框、图片或临时线。优先标注稳定的组/组件端点，避免吸附到易被重建的细分边。模型更新后：

1. 在 LayOut 的“文档设置/引用”中刷新模型引用；
2. 逐页检查断开、偏移、红色警示或落在错误几何上的尺寸；
3. 对照已审核尺寸链与现场复尺，人工签署检查结果。

“引用已更新”不等于关联尺寸仍然正确；尺寸语义和施工可行性必须人工复核。

## 版本与文件名

采用 `<project>_<scheme>_<artifact>_<revision>_<YYYYMMDD>.<ext>`：

- 工作修订：`WIP01`、`WIP02`；
- 对外审阅：`R01`、`R02`；
- 已批准：`A01`；只有完成审批和复尺门禁后才能使用；
- 示例：`A03023_S01_layout_R03_20260720.layout`、`A03023_S01_model_R03_20260720.skp`。

SketchUp、LayOut、manifest、PDF 的 `projectId/schemeId/revision` 必须一致。不得覆盖已发出的修订版；新修订复制后递增编号，并在 manifest 中记录来源、哈希和状态。

## 最少三步交付

插件/桥接只能准备模型、场景、manifest 和待办清单，无法完全自动操作 LayOut。将人工动作收敛为三步：

1. **打开**指定 `.layout`，在“文档设置/引用”刷新对应 `.skp`；
2. **检查**缺失引用、场景、固定比例及全部关联尺寸，签署检查项；
3. **导出**：用户点击 LayOut 的 PDF 导出，核对页数、文件名和 DRAFT/批准状态。

SketchUp Ruby API 不能直接自动化 LayOut 文档完成所有刷新和导出；任何脚本声称“零点击更新”或“自动最终 PDF”都不符合本流程。
